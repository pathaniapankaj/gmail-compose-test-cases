# Gmail Compose Functionality Test Cases

This repository contains QA test cases for Gmail Compose functionality.

## Objective

To validate the Gmail Compose feature and ensure users can successfully create, manage, and send emails while handling positive, negative, and edge-case scenarios.

## Deliverables

* Traditional Test Cases
* BDD Test Cases
* Positive Test Scenarios
* Negative Test Scenarios
* Edge Case Test Scenarios

## Scope

The assessment covers Gmail Compose functionality including:

* Email composition
* Email sending
* Recipient validation
* CC/BCC functionality
* Attachments
* Draft management
* Compose window controls
* Toolbar actions
* Mobile compatibility
* Error handling
* Edge case validation

## Files

* `Traditional Test Cases.xlsx`
* `BDD Test Cases.xlsx`

## Viewing the Test Cases

GitHub does not display Excel (`.xlsx`) file contents directly in the browser.

To view the test cases:

1. Click on the desired Excel file (Traditional Test Cases.xlsx or BDD Test Cases.xlsx).
2. Click View Raw or Download.
3. Open it using Microsoft Excel, Google Sheets, or any compatible spreadsheet application.

## Edge Cases Covered

* Undo Send
* Multiple Compose Windows
* Invalid Recipient Handling
* Sending Without Subject
* Empty Email Content
* Session Expiration During Composition
* Storage Full Behavior
* Large Attachment Handling
* Mobile App Compose Behavior
* Network Interruption During Send

## Test Coverage

The test suite validates:

* Core compose workflow
* Email delivery
* User input validation
* Error handling and recovery
* Cross-platform behavior
* Usability and reliability scenarios
